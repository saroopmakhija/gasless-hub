
OVERVIEW
--------
Gasless Hub combines four core technologies to enable gasless Solana
transactions with cross-chain sponsor deposits from Base:

1. Solana Blockchain (Devnet)
2. Anchor Smart Contract Framework (Fee Pool Program)
3. Kora Paymaster System (Custom Rust Infrastructure)
4. Circle CCTP (Cross-Chain Transfer Protocol)


Technology Stack:
-----------------
• @solana/web3.js (v1.95.8) - Core SDK
• @solana/spl-token (v0.4.9) - SPL Token Program integration
• Native Solana Programs - System Program & Token Program

Key Features:
------------
• Program Derived Addresses (PDAs)
• Associated Token Accounts (ATAs)
• Transaction fee sponsorship via feePayer
• $0.00009 average transaction fee

------------

• RPC Interface: JSON-RPC 2.0 on port 8080

How It Works:
------------
1. Frontend constructs transaction with Kora as feePayer
2. User signs transaction (authorizes operations)
3. Transaction sent to Kora RPC
4. Kora co-signs as fee payer
5. Kora broadcasts to Solana
6. User pays $0 in fees

RPC Methods:
-----------
• getBlockhash - Returns fresh blockhash
• signTransaction - Co-signs without broadcasting
• signAndSendTransaction - Co-signs and submits
• /health - Service health check

Security Model:
--------------
✓ Kora ONLY signs as fee payer (cannot move user funds)
✓ User MUST sign to authorize transfers
✓ Transaction validation before signing

CCTP Flow:
---------
STEP 1: BURN → Sponsor approves USDC, Bridge Kit burns on Base
STEP 2: ATTESTATION → Circle signs burn event off-chain
STEP 3: MINT → USDC minted 1:1 on Solana to Kora's ATA

Why CCTP?
---------
✓ Trust-minimized (no custodians)
✓ 1:1 backing (native USDC, not wrapped)
✓ No liquidity pools (infinite capacity, zero slippage)
✓ $37B+ proven cross-chain volume
✓ Regulatory compliance



Base Sponsor Deposit Flow:
--------------------------
1. Sponsor connects MetaMask/Phantom EVM → Base Sepolia
2. Enters USDC amount → Approves spend
3. Bridge Kit burns USDC → Circle attestation
4. USDC minted to Kora's Solana ATA
5. Fee Pool program records deposit
6. Sponsor appears on leaderboard

Gasless User Transaction Flow:
------------------------------
1. User connects Phantom (no SOL required)
2. Initiates transfer → Frontend constructs tx (Kora as feePayer)
3. User signs → Kora co-signs → Broadcasts
4. Transaction confirmed → User paid $0
